HTML - Tables

HTML tables are used to display data in a structured format of rows and columns. Tables are helpful for organizing information such as data sets, pricing lists, schedules, and more. In this tutorial, you'll learn the basics of creating tables in HTML, how to format them, and some best practices.

Basic Table Structure

An HTML table consists of multiple tags that define its structure:

  • <table>: This tag creates the table itself.
  • <tr>: Defines a row in the table.
  • <th>: Defines a header cell in a table (header cells are typically bold and centered by default).
  • <td>: Defines a standard data cell.

Example

Output

Name Age
John 28
Jane 25

Table Headers

The <th> tag is used for table headers, which are bold and centered by default. These provide context for the data in each column.

Example

Output

Product Price
Laptop $900
Smartphone $600

In this example, "Product" and "Price" are headers that describe the type of data in the columns below them.

Spanning Rows and Columns

HTML allows you to merge cells across multiple rows or columns using the rowspan and colspan attributes.

Column Span (colspan)

You can merge multiple columns into one using colspan.

Row Span (rowspan)

You can merge multiple columns into one using rowspan.

Example

In this example, the "Product" header spans across two rows, covering the data for both "Laptop" and "Smartphone."

Table Sections: <thead>, <tbody>, <tfoot>

Tables can be divided into three sections using <thead>, <tbody>, and <tfoot>. This structure improves readability and makes it easier to style the table.

  • <thead>: Contains the table's header rows.
  • <tbody>: Contains the main body of the table, where most of the data is stored.
  • <tfoot>: Contains the footer of the table, often used for summary rows.

Example

Output

Product Price
Laptop $900
Smartphone $600
Total Products: 2

Best Practices for Using HTML Tables

  • Use Tables for Tabular Data: Tables are designed to organize and display structured data, not for page layout. For layout purposes, use CSS grids or flexbox.
  • Always Use <th> for Headers: Clearly define column or row headers with the <th> tag, which improves accessibility for screen readers.
  • Use Descriptive Captions: If the table contains complex data, use a <caption> to provide context to the user.
  • Keep Tables Responsive: Make tables responsive using CSS techniques, such as setting the table width to 100% and using media queries for mobile optimization.
  • Consider Accessibility: Use the scope attribute in <th> elements (e.g., scope="col" or scope="row") to improve accessibility for users relying on screen readers.
